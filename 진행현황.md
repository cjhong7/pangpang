## 파일 구조

```
goe note/
├── 한글팡팡게임.html       ← 게임 본체 (단일 HTML, 전체 코드)
├── index.html              ← GitHub Pages 배포용 사본 (항상 최신본으로 복사)
├── 게임실행하기.bat         ← 교사용 더블클릭 실행 파일
├── 진행현황.md             ← 이 파일
└── .claude/
    ├── serve.ps1           ← PowerShell HTTP 서버 (포트 8765)
    └── launch.json         ← 미리보기 서버 설정
```

## 구현 완료 기능

### 게임 핵심
- [x] 카메라 배경 (거울 모드, `object-fit:cover`)
- [x] 비눗방울 안에 한글 단어 — 둥근 방울 + 반사광 하이라이트 + 5가지 색조
- [x] 정답 터치 시 방울 터지는 애니메이션 (`bubblePop`)
- [x] 오답 터치 시 빨간 테두리 번쩍임
- [x] 5단계 × 3속도 선택
  - 1단계: 한 글자 +2점 / 2단계: 두 글자 +4점
  - 3단계: 세 글자 +6점 / 4단계: 네 글자 +8점
  - 5단계: 1~4단계 종합 +10점
- [x] 20단어 또는 60초 (먼저 끝나는 것 기준)
- [x] 콤보 카운터 (3콤보 이상 시 화면 표시)
- [x] 성공 효과: 파티클 폭발 + 화면 플래시 + 효과음 (Web Audio API)
- [x] **단어 크기 조정** (`clamp(1.4rem,3.2vw,2.4rem)` — 작게 조정)
- [x] **비눗방울 화면 위 절반에서만 등장** (animationDelay 최대 40%)
- [x] **비눗방울 겹침 방지** (수평 155~195px 동적 간격, getBoundingClientRect 기반)
- [x] **항상 7개 이상 방울 유지** (`MIN_BUBBLES=7`, `fillToBmin()`)
- [x] **속도 고정** (느림 고정, 속도선택 UI 완전 삭제)
- [x] **오답 -1점 페널티** (0점 미만 불가)
- [x] **오답 피드백** — 화면 전체 빨간 오버레이 480ms + 중앙 -1 텍스트 + 버블 빨간 배경 (CSS 애니메이션 없이 순수 DOM 조작)

- [x] 메뉴 화면 손동작 선택 — 700ms 드웰 클릭
- [x] 노란 glow 애니메이션으로 선택 중임을 시각적으로 표시

- [x] 설정에서 On/Off 토글 가능

### 이름 입력
- [x] 음성 인식 (Web Speech API, ko-KR)
- [x] 음성 실패 시 키보드 입력 자동 전환

## 온라인 모드 & 학교 관리

### 접속 방법 (두 가지)
| 방법 | 대상 | 설명 |
|------|------|------|
| 학교명 직접 입력 | 관리자/담임교사 | 학교명 입력 + ☑ 기억하기 옵션 |
| 📲 코드로 참여 | 학생/참여자 기기 | 4글자 참여코드 입력 → GAS 조회 → 자동 설정 적용 |

### 참여코드 (4글자 단축코드 — GAS 연동)
- **GAS 활성 시**: 영어+숫자 **4글자** 코드 (I, O 제외) — 초등학생도 입력 쉬움
  - 관리자가 코드 생성 → GAS "코드" 시트에 저장
  - 학생이 4글자 입력 → GAS에서 학교/범위/URL 조회 → 자동 적용
- **GAS 비활성 시**: 구형 `HGP:` base64 코드 — 하위호환 유지
  - base64url 인코딩 (`+`→`-`, `/`→`_`, `=` 제거)
  - 페이로드 키: `s`=학교, `sc`=범위, `g`=학년, `c`=반, `u`=GAS URL, `p`=PIN해시
- **종류** (공통):
  - 학교전체 코드: 참여자가 직접 학년/반 선택
  - 학년 코드 (1·2·3학년): 해당 학년으로 자동 입장
  - 학년반 코드 (1~10반): 해당 반으로 자동 입장
- **참여코드 적용 기기**: 환경설정(⚙️) 버튼 자동 숨김

### 보안 모델 (모델 A)
- **GAS URL = 학교 신분증**: URL 없이는 학교 데이터에 접근 불가
- **PIN 해시 전파**: 참여코드에 PIN 평문 미포함, 해시만 전파 → 기본 0000 자동승격 차단
- **환경설정 게이트**: 학교명 + 비밀번호 동시 입력 필요 → 학교별 격리
- **참여 기기**: 환경설정 버튼 완전 숨김 (localStorage `hpg_participant=1`)

## 구글 드라이브 연동 (GAS) 설정

### 스프레드시트 시트 구조
- **"점수" 시트** (14컬럼): `학교 / 이름 / 점수 / 학년 / 반 / 단계 / 속도 / 완료단어 / 목표단어 / 소요시간(초) / 정답률(%) / 틀린단어 / 완료여부 / 날짜`
- **"코드" 시트** (8컬럼): `코드 / 학교 / 범위 / 학년 / 반 / GAS_URL / PW해시 / 생성일시` ← **4글자 단축코드 저장**

### 설정 방법 (처음 1회)
1. 학교 구글 계정으로 **drive.google.com** → 새 Google 스프레드시트 생성
2. **확장 프로그램 → Apps Script** 클릭
3. 기존 코드 삭제 → 게임 환경설정에서 **📋 코드 복사** → 붙여넣기
4. `var PASSWORD = "0000";` 을 원하는 비밀번호로 변경 후 💾 저장
5. **배포 → 새 배포 → 웹 앱** → 실행: 나 / 액세스: 모든 사용자 → **배포**
6. 웹 앱 URL 복사 → 환경설정 → 구글 드라이브 연동에 붙여넣기
7. **✅ 사용하기** 체크 → **설정 저장** → **🔌 연결 테스트**

### 데이터 흐름 (GAS 연동 시)
```
참여자 기기: 게임 완료
  → localStorage 저장 (항상)
  → GAS POST → 구글 스프레드시트 (학년/반 정보 포함)

교사 분석 기기: 결과분석 화면 열기
  → GAS GET → 전체/학년/반 bucket 자동 분배 → localStorage 갱신
  → 분석 화면에 모든 참여자 데이터 표시
  → 🔄 동기화 버튼으로 수동 갱신 가능
```
