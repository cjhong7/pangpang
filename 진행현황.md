# 한글 팡팡 게임 — 진행현황

> 마지막 업데이트: 2026-06-09 (세션 5)  
> 제작 목적: 초등 1~3학년 한글 교육용 웹게임 (교실 TV/터치스크린 사용)  
> 배포 URL: https://cjhong7.github.io/pangpang/

---

## 파일 구조

```
pangpang/
├── 한글팡팡게임.html       ← 게임 본체 (단일 HTML, 전체 코드)
├── index.html              ← GitHub Pages 배포용 사본 (항상 최신본으로 복사)
├── 게임실행하기.bat         ← 교사용 더블클릭 실행 파일
├── 진행현황.md             ← 이 파일
└── .claude/
    ├── serve.ps1           ← PowerShell HTTP 서버 (포트 8765)
    └── launch.json         ← 미리보기 서버 설정
```

---

## 실행 방법

**`게임실행하기.bat` 더블클릭** → 서버 창 자동 실행 → Chrome에서 `http://localhost:8765` 자동 오픈

> ⚠️ HTML 파일을 직접 더블클릭하면 카메라·음성이 차단됩니다. 반드시 bat 파일로 실행하세요.  
> ⚠️ 브라우저는 Chrome 권장 (음성인식 가장 안정적)

## 구현 완료 기능

### 게임 핵심
- [x] 카메라 배경 (거울 모드, `object-fit:cover`)
- [x] 비눗방울 안에 한글 단어 — 둥근 방울 + 반사광 하이라이트 + 5가지 색조
- [x] 정답 터치 시 방울 터지는 애니메이션 (`bubblePop`)
- [x] 오답 터치 시 빨간 테두리 번쩍임
- [x] 5단계 (1단계: +2점 / 2단계: +4점 / 3단계: +6점 / 4단계: +8점 / 5단계: +10점)
- [x] 20단어 또는 60초 (먼저 끝나는 것 기준)
- [x] 콤보 카운터 (3콤보 이상 시 화면 표시)
- [x] 성공 효과: 파티클 폭발 + 화면 플래시 + 효과음 (Web Audio API)
- [x] **속도 고정** (느림 고정, 속도선택 UI 완전 삭제)
- [x] **오답 -1점 페널티** (0점 미만 불가)
- [x] **오답 피드백** — 화면 전체 빨간 오버레이 + 중앙 -1 텍스트

### 손동작 인식
- [x] MediaPipe Hands — 검지 끝으로 비눗방울 터치 (게임 중)
- [x] 메뉴 화면 손동작 선택 — 700ms 드웰 클릭
- [x] 거울·cover 좌표 보정 (`videoToScreen`)
- [x] 손 인식 불가 환경 자동 폴백 (화면 터치로 전환)
- [x] 설정에서 On/Off 토글 가능

### 이름 입력
- [x] 음성 인식 (Web Speech API, ko-KR)
- [x] 음성 실패 시 키보드 입력 자동 전환

### 랭킹 (명예의 전당)
- [x] 인트로 화면에 상위 5명 표시
- [x] 모드·학교·범위별 완전 분리 저장 (localStorage)
- [x] 최대 200개 기록 보관 (분석용으로 확장)

### 결과 분석 화면
- [x] **통계 요약**: 총 플레이 횟수 / 참여 학생 수 / 평균 점수 / 평균 정답률 / 평균 시간
- [x] **필터**: 단계별 / 학생별 / 정렬 (최근순·점수순·이름순)
- [x] **select 드롭다운 배경색 수정** (흰 배경에 흰 글씨 문제 해결)
- [x] **엑셀 다운로드 멀티탭**: 전체 / 1학년 / 2학년 / 3학년 / 1학년1반 등 탭 자동 분리
- [x] **클라우드 동기화**: 화면 열릴 때 자동 + 🔄 수동 버튼

### 환경설정 보안
- [x] 학교명 + 관리자 비밀번호 동시 입력 후 잠금 해제
- [x] **환경설정 열 때마다 비밀번호 재입력** (닫으면 잠금 초기화)
- [x] 온라인 재진입 시 isParticipant 플래그 초기화 → 환경설정 버튼 항상 표시
- [x] 참여코드 입장 기기: ⚙️ 버튼 완전 숨김

## 온라인 모드 & 학교 관리

### 접속 방법 (두 가지)
| 방법 | 대상 | 설명 |
|------|------|------|
| 👦 학생 참여 | 교사·관리자·학생 | 학교명 입력 → 범위 직접 선택 |
| 📲 참여코드 | 학생 기기 | 4글자 코드 입력 → GAS 조회 → 자동 설정 적용 |

> ⚠️ 교사 로그인 탭은 제거됨 (2026-06-08)

### 참여코드
- **GAS 활성**: 4글자 코드 (I, O 제외)
- **GAS 비활성**: 구형 `HGP:` base64 코드 (하위호환)
- **재생성 시 경고**: 기존 코드 무효화 안내 확인창 표시
- **종류**: 학교전체 / 학년 / 학년반

### 보안 모델
- **GAS URL = 학교 신분증**: URL 없이는 학교 데이터에 접근 불가
- **드라이브 연동 비밀번호**: 점수 저장 시 인증 (setCode/setPassword는 URL 인증으로 대체)
- **비밀번호 자동 복구**: 불일치 시 자동 재동기화 후 재시도
- **GAS URL 노출 대응**: setPassword/setCode 모두 URL만으로 인증

## 구글 드라이브 연동 (GAS) 설정

### 스프레드시트 시트 구조 (자동 생성)

| 시트명 | 내용 | 컬럼 수 |
|--------|------|--------|
| `전체점수` | 모든 게임 기록 | 14개 |
| `1학년점수` | 1학년 기록 | 14개 |
| `2학년점수` | 2학년 기록 | 14개 |
| `3학년점수` | 3학년 기록 | 14개 |
| `1-1점수` | 1학년 1반 기록 | 14개 |
| `1-2점수` | 1학년 2반 기록 | 14개 |
| ... | (해당 반 플레이 시 자동 생성) | 14개 |
| `코드` | 참여코드 목록 | 8개 |
| `설정` | GAS 비밀번호 등 | 2개 |

> 컬럼(14개): 학교 / 이름 / 점수 / 학년 / 반 / 단계 / 속도 / 완료단어 / 목표단어 / 소요시간(초) / 정답률(%) / 틀린단어 / 완료여부 / 날짜

### 점수 저장 규칙
- 게임 완료 시 `전체점수`에 항상 저장
- 학년 정보 있으면 `N학년점수`에도 저장
- 반 정보 있으면 `N-M점수`에도 저장

### GAS 지원 액션

| 액션 | 메서드 | 설명 |
|------|--------|------|
| (없음) | GET | 전체점수 목록 조회 |
| `getCode` | GET | 참여코드 조회 |
| `setPassword` | POST | 비밀번호 동기화 (인증 불필요) |
| `setCode` | POST | 참여코드 저장 (인증 불필요) |
| `deleteScope` | POST | 범위별 데이터 삭제 (비번 필요) |
| (없음) | POST | 점수 저장 (비번 필요) |

### 설정 방법 (처음 1회)
1. 학교 구글 계정으로 **drive.google.com** → 새 Google 스프레드시트 생성
2. **확장 프로그램 → Apps Script** 클릭
3. 기존 코드 전체 삭제
4. 게임 환경설정 → `📖 설정 방법` 펼치기 → **📋 코드 복사** → 붙여넣기
5. **저장(Ctrl+S)**
6. **배포 → 새 배포 → 웹 앱** → 실행: 나 / 액세스: 모든 사용자 → **배포**
7. 웹 앱 URL 복사 → 환경설정 → 구글 드라이브 연동에 붙여넣기
8. **설정 저장** (비밀번호 GAS 자동 동기화) → **🔌 연결 테스트**

> ⚠️ GAS 코드 변경 후 반드시 **새 배포** (기존 배포 수정 ❌)

### 클라우드 저장 자동 복구
1. 점수 저장 시도
2. 비밀번호 불일치 감지 → 자동 `setPassword` 동기화
3. 점수 저장 재시도
4. 결과 화면에 `✅ 클라우드 저장 완료` 또는 `❌ 오류 내용` 표시
