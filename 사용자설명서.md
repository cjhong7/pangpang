## 1. 게임 소개

**한글 팡팡 게임**은 초등학교 1~6학년을 위한 한글 교육용 웹 게임입니다.
화면에 떠오르는 비눗방울 속 한글 단어 중 **선생님이 제시한 정답 단어**를 찾아
손동작 또는 터치로 터뜨리는 방식입니다.

| 항목 | 내용 |
|------|------|
| 대상 | 초등 1~6학년 |
| 사용 기기 | 교실 터치스크린 TV, 태블릿, PC |
| 게임 시간 | 단계당 최대 60초 (20단어 완료 시 조기 종료) |
| 단계 | 1~5단계 (단계가 높을수록 글자 수 증가, 점수 증가) |
| 입력 방식 | 손동작(MediaPipe) 또는 화면 터치/마우스 클릭 |
| 인터넷 | 오프라인 모드(인터넷 없이 사용 가능) / 온라인 모드(학교별 데이터 관리) |

---

## 2. 접속 방법

### 2-1. 인터넷 브라우저로 접속 (권장)

1. Chrome 브라우저 실행
2. 주소창에 입력: **`https://cjhong7.github.io/pangpang/`**
3. 북마크에 저장해두면 편리합니다.

> ⚠️ **Chrome 브라우저를 권장합니다.** (음성인식 및 카메라 호환성 최적)
> Safari, Edge 등에서는 일부 기능이 제한될 수 있습니다.

### 2-2. 로컬 실행 (인터넷 없이, 교실 PC)

파일이 설치된 PC에서는 `게임실행하기.bat` 파일을 **더블클릭**하면 자동으로 실행됩니다.

> ⚠️ HTML 파일을 직접 더블클릭하면 카메라·음성이 차단됩니다.
> 반드시 **게임실행하기.bat**으로 실행하세요.

## 3. 게임 진행 방법 (학생용)

### 3-1. 시작 화면 — 모드 선택

| 버튼 | 설명 |
|------|------|
| 💻 오프라인으로 시작 | 인터넷 없이, 이 기기에만 기록 저장 |
| ☁️ 학교이름 또는 코드로 시작 | 학교 서버에 점수 저장 (선생님이 분석 가능) |

> 선생님이 **참여코드**를 나눠준 경우 → **☁️ 온라인** 선택

---

### 3-2. 온라인 입장 방법 (두 가지)

#### 방법 A: 학교 이름으로 직접 입장

1. **☁️ 학교이름 또는 코드로 시작** 클릭
2. `👦 학생 참여` 탭에서 **학교 이름 입력** (예: 행복초등학교)
3. 데이터 범위 선택: **학교전체 / 학년 / 반** 중 하나 선택
4. 게임 시작

#### 방법 B: 참여코드로 입장 (권장)

1. **☁️ 학교이름 또는 코드로 시작** 클릭
2. `📲 참여코드` 탭 선택
3. 선생님이 안내한 **4글자 코드** 입력 (예: `ABCD`)
4. 자동으로 범위 설정 후 바로 게임 시작

> ✅ 참여코드를 사용하면 학교 이름을 몰라도 입장할 수 있습니다.
> ✅ 코드는 학교전체 / 학년별 / 반별 세 종류가 있습니다.

---

### 3-3. 이름 입력

- **🎤 마이크 아이콘**이 표시되면 이름을 큰 소리로 말하세요.
- 화면에 이름이 나타나면 **확인** 버튼을 누릅니다.
- 음성 인식이 어려우면 **⌨️ 키보드로 입력** 버튼을 눌러 직접 입력합니다.

---

### 3-4. 단계 선택

| 단계 | 단어 수준 | 정답 점수 |
|------|-----------|-----------|
| 1단계 | 1글자 (예: 산, 달) | +2점 |
| 2단계 | 2글자 (예: 나무, 하늘) | +4점 |
| 3단계 | 3글자 (예: 바나나, 기차역) | +6점 |
| 4단계 | 4글자 (예: 비행기, 무지개) | +8점 |
| 5단계 | 종합 (1~4단계 혼합) | +10점 |

---

### 3-5. 게임 화면

```
┌─────────────────────────────────────────┐
│  ⏱ 남은시간      [단계]      점수       │
│                                         │
│     제시어: [정답 단어]                  │
│                                         │
│   ⭕ 바른    ❌ 틀린    ⭕ 바른          │
│        ⭕ 정답     ❌ 오답               │
└─────────────────────────────────────────┘
```

- **화면 상단**에 찾아야 할 **정답 단어**가 표시됩니다.
- 비눗방울 중 **정답 단어**를 터치하거나 손으로 가리키면 터집니다!
- **오답 터치 시**: 화면이 빨갛게 깜박이고 **-0.5점** 감점
- **정답 터치 시**: 방울이 터지며 **+점수 팝업** 표시, 효과음 재생

> 💡 손동작 사용 시: 검지 손가락 끝을 비눗방울에 **0.5초 이상** 올려두면 반응합니다.

---

### 3-6. 단계 종료 화면

- **총 획득 점수** 확인
- 클라우드 저장 결과 확인 (`✅ 클라우드 저장 완료` 또는 `❌ 연결 실패`)
- 다음 중 선택:
  - **N단계로 이어서 하기** — 다음 단계 계속
  - **새로운 게임자 참여** — 다른 학생 차례
  - **게임 종료** — 인트로 화면으로 돌아가기

## 4. 관리자 초기 설정 (처음 1회)

> 이 설정은 **담당 교사(관리자)** 가 처음 한 번만 하면 됩니다.

### 4-1. 환경설정 열기

1. 게임 시작 후 **온라인 모드**로 학교 이름을 직접 입력해 입장
2. 인트로 화면 우측 상단의 **⚙️ 환경설정** 버튼 클릭
3. **학교명** 과 **관리자 비밀번호** 입력 (처음에는 비밀번호 `0000`)
4. **잠금 해제** 클릭

> ⚠️ 참여코드로 입장한 기기에서는 ⚙️ 버튼이 보이지 않습니다.
> 반드시 **👦 학생 참여** 탭으로 학교명 직접 입력 후 입장하세요.

---

### 4-2. 학교 정보 설정

환경설정 → **학교 설정** 섹션:

| 항목 | 설명 |
|------|------|
| 학교 이름 | 학교명 (학생들이 입장할 때 입력하는 이름과 동일하게) |
| 관리자 비밀번호 | 환경설정 잠금 해제용 (기본값: `0000`, 변경 권장) |
| GAS 웹앱 URL | 구글 드라이브 연동 주소 (다음 섹션 참조) |
| 드라이브 연동 비밀번호 | 점수 저장 인증용 (기본값: `0000`) |

**설정 저장** 버튼을 눌러 저장합니다.

---

### 4-3. 비밀번호 변경 방법

1. 환경설정 열기 (현재 비밀번호로 잠금 해제)
2. **관리자 비밀번호** 칸에 새 비밀번호 입력
3. **설정 저장** 클릭

> 💡 비밀번호를 잊어버린 경우: 브라우저 개발자 도구(F12) → Console에서
> `localStorage.removeItem('hpg_pw_admin_학교명')` 실행 후 새로 설정

## 5. 구글 드라이브 연동 설정 (GAS)

> **이 설정은 선택사항**입니다. 연동하지 않아도 오프라인으로 사용 가능합니다.
> 연동하면 여러 기기에서 같은 학교 점수를 공유하고 교사가 결과를 분석할 수 있습니다.

### 5-1. 사전 준비

- 학교 구글 계정 (또는 개인 구글 계정)
- Chrome 브라우저

---

### 5-2. Google 스프레드시트 생성

1. **drive.google.com** 접속 (학교 구글 계정으로 로그인)
2. **+ 새로 만들기 → Google 스프레드시트** 클릭
3. 파일 이름 설정 (예: `한글팡팡 점수 관리`)
4. 스프레드시트가 열리면 다음 단계 진행

---

### 5-3. Apps Script 코드 설치

1. 스프레드시트 메뉴 → **확장 프로그램 → Apps Script** 클릭
   - 새 탭에서 스크립트 편집기가 열립니다.

2. 기존 코드 전체 선택 후 삭제 (`Ctrl+A` → `Delete`)

3. **한글팡팡게임 → 환경설정** 열기 → 맨 아래 **📖 설정 방법** 섹션 펼치기 → **📋 코드 복사** 클릭
   - 클립보드에 GAS 코드가 복사됩니다.

4. Apps Script 편집기에 붙여넣기 (`Ctrl+V`)

5. **저장** (`Ctrl+S`) 후 프로젝트 이름 입력 (예: `한글팡팡`)

---

### 5-4. 웹 앱 배포

1. Apps Script 편집기 우측 상단 **배포 → 새 배포** 클릭

2. ⚙️ 아이콘 클릭 → **웹 앱** 선택

3. 다음과 같이 설정:

   | 항목 | 설정값 |
   |------|--------|
   | 설명 | (자유롭게 입력, 예: `한글팡팡 v1`) |
   | 실행 계정 | **나** |
   | 액세스 권한 | **모든 사용자** ← 반드시 이 설정 |

4. **배포** 클릭 → 권한 요청 창이 뜨면 **액세스 허용**

5. 배포 완료 후 **웹 앱 URL** 복사 (예: `https://script.google.com/macros/s/AKfycb.../exec`)

---

### 5-5. 게임에 URL 등록

1. 환경설정 열기 → **구글 드라이브(스프레드시트) 연동** 섹션

2. **웹 앱 URL 칸**에 복사한 URL 붙여넣기

3. **드라이브 연동 비밀번호** 설정 (기본 `0000`, 변경 가능)

4. **사용하기** 체크박스 ✅ 체크

5. **설정 저장** 클릭

6. **🔌 연결 테스트** 클릭 → `✅ 연결 성공!` 확인

---

### 5-6. GAS 코드 업데이트 방법 (기능 추가 시)

새 기능이 추가되면 GAS 코드도 함께 업데이트해야 합니다:

1. 환경설정 → **📋 코드 복사**
2. Apps Script 편집기 열기 → 기존 코드 전체 삭제 → 붙여넣기 → 저장
3. **배포 관리 → ✏️ 수정 → 버전: 새 버전 선택 → 배포**

> ⚠️ **"새 버전"** 으로 배포해야 합니다. 기존 버전을 그대로 두면 코드가 반영되지 않습니다.
> URL은 변경되지 않으므로 게임에 다시 등록할 필요 없습니다.

---

### 5-7. 스프레드시트 자동 생성 시트

게임이 진행되면 스프레드시트에 다음 시트들이 자동으로 생성됩니다:

| 시트명 | 내용 |
|--------|------|
| `전체점수` | 학교 전체 게임 기록 |
| `1학년점수` | 1학년 전체 기록 |
| `2학년점수` | 2학년 전체 기록 |
| `1-1점수` | 1학년 1반 기록 |
| `1-2점수` | 1학년 2반 기록 |
| `코드` | 발급된 참여코드 목록 |
| `설정` | 관리자 비밀번호 등 |

> 💡 각 시트에는 **학교 / 이름 / 점수 / 학년 / 반 / 단계 / 완료단어 / 정답률 / 소요시간 / 날짜** 등 14개 항목이 기록됩니다.

## 6. 참여코드 발급 및 배포

> 참여코드를 발급하면 학생들이 학교명을 몰라도 4글자 코드만으로 게임에 입장할 수 있습니다.

### 6-1. 참여코드 종류

| 코드 종류 | 입장 범위 | 사용 예시 |
|-----------|-----------|-----------|
| 학교전체 코드 | 학교 전체 학생 | 전교생 대상 게임 |
| 학년 코드 | 특정 학년 학생 | 3학년 전체 대상 |
| 반별 코드 | 특정 반 학생 | 2학년 1반 대상 |

---

### 6-2. 학교전체 참여코드 발급

1. 환경설정 → **학교전체 관리** 섹션
2. **🔗 참여코드 생성** 클릭
3. 4글자 코드가 자동 생성됩니다 (예: `UQNW`)
4. **복사** 버튼으로 클립보드에 복사하거나 **QR** 버튼으로 QR코드 생성

---

### 6-3. 학년별 참여코드 발급

1. 환경설정 → **학년별 관리** 섹션
2. 상단 버튼에서 학년 선택 (가.1학년 ~ 바.6학년)
3. **🔗 참여코드 생성** 클릭
4. 코드 복사 또는 QR 생성

---

### 6-4. 반별 참여코드 발급

1. 환경설정 → **학년반별 관리** 섹션
2. 학년(1~6학년) 블록에서 반 버튼 클릭 (1반~10반)
3. **🔗 참여코드 생성** 클릭
4. 코드 복사 또는 QR 생성

> 💡 **QR 코드 활용법**: QR 이미지를 화면에 띄워 학생들이 스마트폰으로 스캔하면 바로 입장됩니다.

---

### 6-5. 코드 재발급 안내

이미 발급된 코드가 있을 때 다시 생성하면:
- `"이전 코드는 사용할 수 없습니다. 그래도 진행하시겠습니까?"` 확인창 표시
- **확인** 선택 시 기존 코드 무효화 + 새 코드 발급
- **취소** 선택 시 기존 코드 유지

> ⚠️ 이미 학생에게 배포한 코드가 있다면 재발급 전에 학생들에게 새 코드를 안내해야 합니다.

## 7. 환경설정 메뉴 전체 안내

## 9. 손동작 인식 사용법

### 9-1. 작동 원리

- 카메라로 **검지 손가락 끝**을 추적합니다.
- 비눗방울 위에 검지를 **0.5초 이상** 올려두면 터치로 인식됩니다.
- 빠르게 스치고 지나가면 반응하지 않습니다.

### 9-2. 최적 사용 환경

| 항목 | 권장 |
|------|------|
| 조명 | 밝은 실내 (역광 X) |
| 배경 | 단순한 배경 |
| 거리 | 카메라에서 50cm ~ 1.5m |
| 손 방향 | 손바닥이 카메라를 향하도록 |

### 9-3. 손 인식이 안 될 때

- 3초 이상 손이 화면에 없으면 **자동으로 카메라 재시작**
- 그래도 안 될 경우 환경설정 → **손동작 인식 On/Off** 체크 해제 후 재체크
- 조명이 어두우면 Off로 설정하고 화면 터치 사용

## 10. 자주 묻는 질문 (FAQ)

### Q1. 참여코드를 입력했는데 "코드를 찾을 수 없습니다" 가 뜨면?

**원인:**
1. 코드 입력 오류 (O와 0, I와 1 혼동 주의 — 게임은 I, O를 코드에 사용하지 않음)
2. 관리자가 GAS를 아직 배포하지 않은 상태
3. GAS URL이 환경설정에 등록되지 않은 상태

**해결:** 환경설정 → 🔌 연결 테스트로 GAS 연결 상태 확인

---

### Q2. "구글 드라이브 연결 실패" 메시지가 나타나면?

**원인:**
1. GAS 스크립트가 배포되지 않았거나 URL이 잘못됨
2. 일시적인 Google 서버 오류

**해결:**
1. 환경설정 → GAS URL 확인
2. 🔌 연결 테스트 클릭
3. 해결 안 되면 GAS 재배포 (배포 관리 → 새 버전)
4. 잠시 후 다시 시도 (일시적 오류인 경우 자동 복구)

---

### Q3. 음성 인식으로 이름을 말했는데 이름에 마침표(.)가 붙어요.

게임이 자동으로 마침표를 제거합니다. 그래도 이상한 경우 **⌨️ 키보드로 입력** 버튼을 이용하세요.

---

### Q4. 점수가 저장되지 않아요.

**체크 사항:**
1. 온라인 모드인지 확인 (오프라인 모드는 클라우드에 저장되지 않음)
2. GAS 연동이 설정되어 있는지 확인
3. 게임 종료 화면에서 `✅ 클라우드 저장 완료` 메시지 확인

---

### Q5. 환경설정 비밀번호를 잊었어요.

1. 브라우저 개발자 도구 열기 (F12 또는 Ctrl+Shift+I)
2. **Console** 탭 클릭
3. 다음 명령어 입력 (학교명에 실제 학교명 입력):

   ```javascript
   localStorage.removeItem('hpg_pw_admin_학교명')
   localStorage.removeItem('hpg_pwh_학교명')
   ```

4. 페이지 새로고침 후 비밀번호 `0000` 으로 다시 설정

---

### Q6. 손동작이 너무 예민하거나 반응이 없어요.

- **너무 예민함** (조금만 닿아도 반응): 손을 좀 더 천천히 움직이세요. 0.5초 이상 유지해야 함
- **반응 없음**: 카메라 앞에 손이 잘 보이는지 확인. 조명 개선 필요할 수 있음
- 손동작이 불편하면 환경설정에서 **손동작 인식 Off** → 터치/마우스로 이용

---

### Q7. 기존 데이터는 유지하면서 GAS 코드만 업데이트할 수 있나요?

**가능합니다.** GAS 코드를 새 버전으로 배포해도 스프레드시트 데이터는 삭제되지 않습니다.
코드만 교체 후 배포하면 됩니다.

---

### Q8. 여러 반이 동시에 사용해도 되나요?

**네.** 반별 참여코드를 각각 발급하면 데이터가 반별로 분리됩니다.
여러 기기에서 동시에 다른 코드로 접속해도 데이터가 섞이지 않습니다.

---

### Q9. 오프라인으로 사용 중인데 나중에 온라인으로 바꿀 수 있나요?

오프라인 기록은 온라인 클라우드로 자동 이전되지 않습니다.
온라인으로 전환하면 새로운 기록부터 클라우드에 저장됩니다.

---

### Q10. 엑셀 단어 등록이 안 될 때?

엑셀 파일 헤더(첫 번째 행) 확인:
- ✅ 올바른 예: `1단계`, `2단계`, `3단계`, `4단계`, `5단계`
- ❌ 잘못된 예: `1단계:`, `1 단계`, `Stage1` 등

헤더 이름이 정확히 맞지 않으면 단어가 로드되지 않습니다.
